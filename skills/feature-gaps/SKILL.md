# Feature Gaps (Deprecated)

`$feature-gaps` preserves the legacy workflow while new prompts migrate to the
explicit names:

- For `Start <ID>`, `Approved <ID>-N`, `Done <ID>`, or `Implement
  $feature-gaps`, read and follow `../feature-gaps-implement/SKILL.md` completely.
- For Find-mode requests and all other `$feature-gaps` invocations, read and
  follow `../feature-gaps-find/SKILL.md` completely.

The selected canonical skill owns the workflow's scope, ledger contract,
delegation, and approval gates.
