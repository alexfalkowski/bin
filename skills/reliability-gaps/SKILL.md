# Reliability Gaps (Deprecated)

`$reliability-gaps` preserves the legacy workflow while new prompts migrate to
the explicit names:

- For `Start <ID>`, `Approved <ID>-N`, `Done <ID>`, or `Implement
  $reliability-gaps`, read and follow
  `../reliability-gaps-implement/SKILL.md` completely.
- For Find-mode requests and all other `$reliability-gaps` invocations, read and
  follow `../reliability-gaps-find/SKILL.md` completely.

The selected canonical skill owns the workflow's scope, ledger contract,
delegation, and approval gates.
